# Minimizer-Based Sketching: ENCODE Validation Experiments

## Project Summary

This project validates minimizer-based sequence sketching using publicly available ENCODE and Roadmap Epigenomics ChIP-seq data through two complementary experiments:

**Experiment A — Cross-Reference Robustness**  
The same biological sample aligned to GRCh37 and GRCh38 should produce more similar sketches than sketches from a different biological sample aligned to either reference.

**Experiment B — Tissue-over-Species Clustering**  
Minimizer sketch similarity should reproduce the well-established result that epigenomic profiles cluster by tissue type rather than by species when comparing matched human and mouse samples. This recapitulates findings from Yue et al. (2014), Lin et al. (2014), and the Roadmap Epigenomics Consortium (2015).

## Quickstart

### 1. Resolve accession numbers

```bash
pip install pysradb requests pandas
python scripts/fetch_accessions.py
# Review config/confirmed_accessions.tsv
# Update config/config.yaml sra_map with confirmed SRR IDs
```

### 2. Run with Snakemake (local)

```bash
cd minimizer-encode-project/
snakemake --cores 8 --use-singularity \
  --singularity-args "--bind /scratch,/ref"
```

### 3. Run on SLURM cluster

```bash
snakemake --profile workflow/slurm_profile/ \
  --use-singularity \
  --singularity-args "--bind /scratch,/ref"
```

### 4. Dry run (check DAG without executing)

```bash
snakemake --dryrun --forceall
snakemake --dag | dot -Tpng > dag.png
```

## Configuration

Edit `config/config.yaml` before running:

- `sketch_bin`: path to your minimizer sketching binary
- `references`: paths to reference FASTA files on your cluster
- `outdir` / `scratch`: output and scratch directory paths
- `sra_map`: SRR run IDs (populated by `fetch_accessions.py`)

## Compute Requirements

| Step | CPUs | RAM | Est. walltime |
|---|---|---|---|
| Download (per sample) | 4 | 8 GB | ~2 h |
| Align (BWA-MEM2, per sample) | 16 | 32 GB | ~4 h |
| Sketch (per BAM) | 4 | 16 GB | ~1 h |
| Similarity matrix | 8 | 16 GB | ~30 min |
| R analysis | 2 | 8 GB | <10 min |

Alignment and sketching benefit from the `bigmem` partition (configured in `slurm_profile/config.yaml`).
